4.寻找两个正序数组的中位数

Posted on 2021-11-30 | In LeetCode

题目

给定两个大小分别为 m 和 n 的正序（从小到大）数组 nums1 和 nums2。请你找出并返回这两个正序数组的 中位数 。

算法的时间复杂度应该为$ O(log (m+n))$ 。

示例 1：

输入：nums1 = [1,3], nums2 = [2]

输出：2.00000

解释：合并数组 = [1,2,3] ，中位数 2

示例 2：

输入：nums1 = [1,2], nums2 = [3,4]

输出：2.50000

解释：合并数组 = [1,2,3,4] ，中位数 (2 + 3) / 2 = 2.5

示例 3：

输入：nums1 = [0,0], nums2 = [0,0]

输出：0.00000

示例 4：

输入：nums1 = [], nums2 = [1]

输出：1.00000

示例 5：

输入：nums1 = [2], nums2 = []

输出：2.00000

提示：

  • \(nums1.length == m\)
  • \(nums2.length == n\)
  • \(0\leq m\leq 1000\)
  • \(0\leq n\leq 1000\)
  • \(1\leq m + n\leq 2000\)
  • \(-10^6\leq nums1[i], nums2[i]\leq 10^6\)

思路

由题目要求的时间复杂度可知，用二分法

假设，我们要找的中位数是合并后的第\(k\)个数，比较\(A[k/2-1]\)和\(B[k/2-1]\)，前面都有\(k/2-1\)个数

  • 如果 \(A[k/2-1] < B[k/2-1]\)，则比\(A[k/2-1]\) 小的数最多只有$A \(的前\)k/2−1$个数和 \(B\) 的前 \(k/2−1\)个数，即比\(A[k/2−1]\)小的数最多只有\(k−2\)个，因此\(A[k/2−1]\)不可能是第\(k\)个数,\(A[0]\)到 \(A[k/2−1]\)也都不可能是第\(k\)个数，可以全部排除。

  • 如果 \(A[k/2-1] > B[k/2-1]\)，同上

  • 如果 \(A[k/2-1] = B[k/2-1]\)，任意删去\(A\)或\(B\)的其中一个即可

有三种特殊情况

  • \(k/2-1\)可能大于数组的边界，所以每次更新时要进行判断，取小的那个。而且，如果刚好越界了，就只能取数组的边界了，这时进行删除时，就不能直接减去\(k/2\)，所以每次减去时，应该用其他变量来求个数
  • 如果其中一个数组空了，就说明那个数组中的所有元素都排除完了，\(k\)也跟着更新了，可以直接返回另一个数组中的第\(k\)个元素
  • 如果\(k=1\)，说明下一个元素就是要找的元素，比较两个数组中当前位置哪个元素较小即可

定点整数的运算

补码加减运算公式

\([X+Y]_补=[X]_补+[Y]_补\quad (mod\quad 2^{n+1})\)

\([X-Y]_补=[X]_补+[-Y]_补\quad (mod\quad 2^{n+1})\)

若\([Y]_补=Y_0,Y_1\cdots Y_n\)，则\([-Y]_补=\overline Y_0\overline Y_1\cdots \overline Y_n+1\)

若\([Y]_补=Y_0.Y_1\cdots Y_n\)，则\([-Y]_补=\overline Y_0\overline Y_1\cdots \overline Y_n+0.0\cdots01\)

定点数加法/减法的运算方法的溢出检测判断

进位判溢方法

\(V=C_1\oplus C_f\)

其中\(C_1\)为最高有效位进位，\(C_f\)为符号位进位

当\(V\)为\(1\)时，即\(C_1\)、\(C_f\)不相同时，溢出

双符号位判溢方法

\(V=S_{f1}\oplus S_{f2}\)

当\(V\)为\(1\)时，即\(S_{f1}\)、\(S_{f2}\)不相同时，溢出

当\(S_{f1}S_{f2}=01\)时，正溢出

当\(S_{f1}S_{f2}=10\)时，负溢出

逻辑位移

将数据视为\(无符号数据\)

逻辑左移，高位移出，低位补\(0\)

逻辑右移，低位移出，高位补\(0\)

算术移位

带符号的数据，符号位必须保持不变

原码

算术左移，符号位不变，高位移出，低位补\(0\)

算术右移，符号位不变，低位移出，高位补\(0\)

补码

算术左移，符号位不变，高位移出，低位补\(0\)（\(符号位\)和\(最高有效位\)不相同时溢出）

算术右移，符号位不变，低位移出，高位补\(符号位\)

反码

算术左移，高位移出，低位补\(符号位\)

算术右移，低位移出，高位补\(符号位\)

定点整数无符号阵列乘法器的原理

\(m\)位\(\times\)\(n\)位绝对值阵列乘法器共需要\(n\times m\)个与门，\(n\times (m-1)\)个全加器

定点运算器的组成与结构

三种不同总线结构下定点运算器的运算步骤

单总线结构

结构一

  • \((R_i)\rightarrow LA:通过发送信号R_i\rightarrow IB、IB\rightarrow LA来实现\)
  • \((R_j)\rightarrow LB:通过发送信号R_j\rightarrow IB、IB\rightarrow LB来实现\)
  • \(ALU运算，结果\rightarrow R_j;通过发送信号C\_OP(\theta)、ALU\rightarrow IB、IB\rightarrow R_k来实现\)

结构二

  • \((R_i)\rightarrow LA:通过发送信号R_i\rightarrow IB、IB\rightarrow LA来实现\)
  • \((R_j)\rightarrow IB，ALU运算，结果\rightarrow LC：通过发送信号R_j\rightarrow IB、C\_OP(\theta)、ALU\rightarrow LC来实现\)
  • \((LC)\rightarrow R_k:通过发送信号LC\rightarrow IB、IB\rightarrow R_k来实现\)

双总线结构

结构一

  • \((R_i)\rightarrow IB1，(R_j)\rightarrow IB2，ALU运算，结果\rightarrow LC：通过发送信号R_i\rightarrow IB1、R_j\rightarrow IB2、C\_OP(\theta)、\\ALU\rightarrow LC来实现\)
  • \((LC)\rightarrow R_k：通过发送信号LC\rightarrow IB1、IB1\rightarrow R_k来实现\)

结构二

  • \((R_i)\rightarrow LA：通过发送信号R_i\rightarrow IB1、IB1\rightarrow LA来实现\)
  • \((R_j)\rightarrow IB2，ALU运算，IB1\rightarrow R_k：通过发送信号R_j\rightarrow IB2、C\_OP(\theta)、ALU\rightarrow IB1、IB1\rightarrow R_k来实现\)

三总线结构

  • \((R_i)\rightarrow IB1，(R_j)\rightarrow IB2，ALU运算，IB3\rightarrow R_k：需要发送信号R_i\rightarrow IB1、R_j\rightarrow IB2、C\_OP(\theta)、\\ALU\rightarrow IB3、IB3\rightarrow R_k\)

定点运算器结构中暂存器 LA、LC 和缓冲器的作用及更换后的工作过程判断

  • 缓冲器、LA更换为LC，可以正常工作，因为LC可以控制输出
  • 缓冲器更换成LA，不能正常工作，因为LA会一直占用数据总线

自定义规格化表示的浮点数运算的5个经典步骤

1.\(0\)操作数检查

2.对阶

  • 比较两个数的阶码\(E_X\)和\(E_Y\)是否相等，即求\(\Delta E=E_X-E_Y\)，若\(\Delta E=0\)，则\(E_X=E_Y\)，若\(\Delta E\neq 0\)，则\(E_X\neq E_Y\)，要进行对阶操作
  • 若\(\Delta E>0\)，则\(E_X>E_Y\)，\(M_Y\)每右移一位，\(E_Y+1\)，直至\(E_X=E_Y\)
  • 若\(\Delta E<0\)，则\(E_X<E_Y\)，\(M_X\)每右移一位，\(E_X+1\)，直至\(E_X=E_Y\)
  • 小阶向大阶对齐，原因：小阶对大阶时，右移小阶的尾数舍去的是尾数的低位。大阶对小阶时，必须左移大阶的尾数，舍去的是尾数的高位。即小阶对大阶更能保证精度

3.尾数相加减

4.结果规格化

情况一

情况二

当\(符号位=最高有效位\)时，左规，直到不相等，每移一位，阶码减一

阶码上溢时，报错，阶码下溢时，将浮点数视为机器零

用\(path\)记录路径，每行的最后一个不为\(0\)的元素下标即是花瓶的序号,因为每层是从左往右遍历的

动态规划

状态转移方程可以是，当前状态=max/min{选择时的上一个状态+状态转移量，不选时的上一个状态}

解题思路

这是一道求单源最短路径的题目，可以利用dijkstra算法求解

但是题目增加了限制条件，当两个路口的交通灯颜色相同时才能通过，可能会存在两路口的交通灯颜色一直都不会相同的情况，所以要进行判断，并且等待的时间也要计算在内

dijkstra算法

假设，起点为A，先将A加到S集合中

由A点一步可以到达的点有：B,C;

A->B:3

A->C:2

从中选一个长度最小的：A->C

将C加入到S集合中

在以C为起点，找最小的边：C->B

将A->B的长度和A->C+C->B的长度比较，后者小的话就更新

重复上述步骤，直到所有点都加到S集合中

3.1 定点整数的机器数

机器数

数值数据在计算机中的表示形式称为\(机器数\)

特点

1.数值范围受计算机字长的限制

2.符号位用二进制的0或1表示

3.小数点用规定的隐含方式表达

原码、反码、补码、移码的定义

原码

最简单、最接近真值原型的一种表示方法

正号用\(0\)表示，负号用\(1\)表示，数值部分与\(绝对值\)一致

数学定义及计算方法

\[ [X]_{原}=\begin{cases} 0X&X\ge0 \\ 1X& X\le0 \end{cases} \]

特点

真值\(0\)的原码表示不唯一，它占用了两个二进制编码

\([+0]_{原}=000\cdots0\)

\([-0]_{原}=100\cdots0\)

反码

通常用来实现转换中的过度

数学定义及计算方法

\[ [X]_{反}=\begin{cases} 0X&X\ge0 \\ 1\overline{X}&X\le0 \end{cases} \]

特点

真值\(0\)的原码表示不唯一，它占用了两个二进制编码

\([+0]_{反}=000\cdots0\)

\([-0]_{反}=111\cdots1\)

补码

最广泛应用，能将减法用加法实现

正号用\(0\)表示，负号用\(1\)表示，数值部分与\(原码\)不同

数学定义及计算方法

\[ [X]_{补}=\begin{cases} 0X&X\ge0 \\ 1\overline{X}+1(末位)&X<0 \end{cases} \]

特点

\([+0]_{补}=[-0]_{补}=000\cdots0\)

可看成只有\(+0\)，\(-0\)的编码用于表示最小的值\(-2^n\)或\(-1\)

移码

主要用于浮点机器数的\(阶码\)

正号用\(1\)表示，负号用\(0\)表示，数值部分与\(补码\)相同

数学定义及计算方法

\[ [X]_{移}=\begin{cases} 1X&X\ge0 \\ 0\overline{X}+1(末位)&X<0 \end{cases} \]

特点

\([+0]_{补}=[-0]_{补}=100\cdots0\)

可看成只有\(+0\)，\(-0\)的编码用于表示最小的值\(-2^n\)或\(-1\)

原码、反码、补码、移码描述的真值范围

原码（n+1)位机器数

定点整数

\(-(2^n-1)\le X \le2^n-1\)

定点小数

\(-(1-2^{-n})\le X \le1-2^{-n}\)

反码（n+1)位机器数

定点整数

\(-(2^n-1)\le X \le2^n-1\)

定点小数

\(-(1-2^{-n})\le X \le1-2^{-n}\)

补码（n+1)位机器数

定点整数

\(-2^n\le X \le2^n-1\)

定点小数

\(-1\le X \le1-2^{-n}\)

与\(原码\)相比，\(补码\)表示的整数和小数范围在负向分别多了一个值：\(-2^n\)和\(-1\)

因为真值\(0\)只占用一个编码\(000\cdots0\)，另外一个在\(原码\)中表示\(-0\)的编码\(100\cdots0\)则用来表示值\(-2^n\)和\(-1\)(最小的数)

移码（n+1)位机器数

与\(补码\)相同

3.2 浮点数的机器数

IEEE754标准32位

阶码为移码，偏移值要减\(1\)

尾数为原码

格式

\(M_{s}\) \(E_{s}\) \(E_{1}E_{2}\cdots E_{m}\) \(.\) \(M_{1}M_{2}\cdots M_{n}\)

数符 阶符 阶码数值 尾数数值

尾数24位（含1位数符），阶码8位（含1位阶符）

真值计算

\(N=(-1)^{M_{s}}\times (1.M_{1}M_{2}\cdots M_{n})\times 2^{E-127}\)

其中\(E\)为阶码的无符号真值

自定义格式下浮点数规格化的表示方法

原码

原码表示的尾数，当最高有效位\(（M_{1}）\)为\(1\)时，浮点数规格化，即尾数为\(X.1X\cdots X\)

补码

补码表示的尾数，当符号位\(（M_{S}）\)与最高有效位\(（M_{1}）\)相异时，浮点数规格化，即尾数为\(0.1X\cdots X\)或\(1.0X\cdots X\)

当符号位与最高有效位相同时，必须左规

自定义格式下浮点数规格化、非规格化表示，描述的浮点数真值范围

阶码为移码，尾数为补码

规格化表示范围

最小数 \(-1\times2^{31}=-2^{31}\) \(1,11111\quad 1.000000000\)

最大负数 \(-(2^{-1}+2^{-9}\times2^{-32})\) \(0,00000\quad 1.011111111\)

最小正数 \(2^{-1}\times2^{-32}=2^{-33}\) \(0,00000\quad 0.100000000\)

最大数 \((1-2^{-9}\times2^{31})\) \(1,11111\quad 0.111111111\)

非规格化表示范围

最小数 \(-1\times2^{31}=-2^{31}\) \(1,11111\quad 1.000000000\)

最大负数 \(-2^{-9}\times2^{-32}=-2^{-41}\) \(0,00000\quad 1.111111111\)

最小正数 \(2^{-9}\times2^{-32}=2^{-41}\) \(0,00000\quad 0.000000001\)

最大数 \((1-2^{-9}\times2^{31})\) \(1,11111\quad 0.111111111\)

若不存在，就随便去一个度数大于0的定点开始递归
